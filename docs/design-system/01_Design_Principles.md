# PIA Design Principles

## Product Character

PIA should feel like a compact institutional command surface for private investing.

- Mobile-first.
- Dense but readable.
- Dark, premium, glass/neon restraint.
- Fast to scan.
- Calm under volatility.
- Built for repeated daily use.

## Mobile-First Principles

- One-hand reach matters more than desktop symmetry.
- No horizontal overflow.
- No clipped cards.
- No dead controls.
- No oversized admin-table controls on mobile.
- Bottom sheets, rails, chips, segmented controls, and compact cards are preferred.
- Content must remain useful when privacy mode masks values.

## Desktop Principles

- Desktop is the power workspace, not a separate product.
- Preserve the same information hierarchy as mobile.
- Use larger canvases for comparison and multi-panel workflows.
- Do not add marketing-style hero layouts inside the app.

## Decision Hierarchy

When tradeoffs conflict, choose in this order:

1. Route integrity and privacy mode.
2. Mobile usability.
3. Scannability and density.
4. Design consistency.
5. Desktop polish.

## Redesign Rule

No developer may redesign freely without an approved mock reference. If the current UI seems wrong, create a mock first and mark it DRAFT.
