# Tokens, Colors, Typography

## Color Direction

PIA uses dark institutional surfaces with restrained blue, green, amber, red, and violet accents.

Core semantic roles:

- Background: near-black radial or flat dark.
- Panel: dark navy/charcoal.
- Border: low-contrast slate.
- Text: cool white.
- Muted text: blue-gray.
- Success: green.
- Warning: amber.
- Danger: red.
- Information: blue.
- AI/decision accent: violet or blue-violet.

## Usage Rules

- Do not create a one-note palette dominated by one hue.
- Do not use decorative gradient blobs or orbs.
- Use gradients sparingly for active states, not as a substitute for hierarchy.
- Risk and loss states must use danger red consistently.
- Positive performance and healthy status must use success green consistently.
- Warnings and placeholder/mock states must use amber.

## Typography

- Use compact, high-legibility system typography.
- Do not scale font size with viewport width.
- Letter spacing must remain `0` unless used for small uppercase labels.
- Large display type belongs only to true top-level page titles or hero metrics.
- Compact panels, cards, sheets, and controls must use tighter type sizes.

## Numeric Display

- Financial values must align with privacy mode.
- Percentages and money values should be scan-friendly.
- Avoid long uncontrolled strings in chips, buttons, and table cells.
- Use truncation or wrapping rules based on the approved mock.
