# Mobile Mocks Locked

## Scope

This file tracks locked mobile mock rules. Actual mock assets live under `docs/design-system/mocks/mobile/` and related category folders.

## Mobile Mock Requirements

Every mobile mock must show:

- Top app/header state.
- Primary content state.
- Bottom navigation or sheet state where relevant.
- Privacy mode behavior.
- Empty/demo fallback state where data may be unavailable.
- One-hand reach assumptions.
- Overflow handling.

## Locked Mobile Rules

- Notification Center must be a bottom sheet.
- Notification cards must stack vertically.
- Scanner sort and filter controls must use compact chip rails.
- Market Pulse and similar rows must use swipe rails.
- Portfolio Snapshot redesign requires approved mobile mock first.
- Quick controls must remain visible and thumb-friendly.

## Approval Register

No mobile redesign is approved unless a mock in `docs/design-system/mocks/mobile/` or the relevant category folder is marked APPROVED.
