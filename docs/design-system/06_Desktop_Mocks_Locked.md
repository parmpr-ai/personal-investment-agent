# Desktop Mocks Locked

## Scope

Desktop mocks live under `docs/design-system/mocks/desktop/` and category folders.

## Desktop Rules

- Desktop follows mobile-approved hierarchy.
- Desktop may add comparison density, not different product meaning.
- Desktop cannot introduce navigation or widget behavior not represented in mobile governance unless explicitly approved.
- Desktop routes must preserve `/`, `/mobile`, and `/setup`.

## Required Desktop Mock Content

- Route or workspace context.
- Grid structure.
- Widget hierarchy.
- Empty/loading behavior.
- Privacy mode behavior.
- Interaction notes.

## Approval Register

Desktop UI implementation requires an APPROVED mock unless the change is non-visual or fixes a bug without altering layout.
