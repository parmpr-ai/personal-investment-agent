# PIA Design System Read Me

This folder stores product-approved design direction, governance, and mock specifications for PIA.

## Operating Rules

- Create mocks/specs before implementing major UI corrections.
- Keep mobile-first density, privacy mode, and route continuity intact.
- Prefer IBKR-inspired terminal ergonomics: compact rows, dense metrics, fast scanning, restrained dark surfaces.
- Do not introduce final UI changes from mock-pack tasks until Product Owner review approves the direction.
- Keep mock specs actionable enough for implementation agents: layout, hierarchy, interaction, states, acceptance criteria, and non-goals.

- `mocks/mobile/`: Sprint 2C mobile correction mock pack for Product Owner review.
